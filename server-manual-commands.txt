MANUAL SERVER COMMANDS TO FIX GIT CONFLICT
==========================================

Run these commands directly on the server (ubuntu@54.169.107.64):

1. Navigate to project directory:
   cd ~/inventoryfullstack

2. Stash local changes to avoid conflict:
   git stash

3. Pull latest code from GitHub:
   git pull origin main

4. Update database schema:
   sudo mysql inventory_db < fix-audit-logs-schema.sql

5. Kill existing node processes:
   sudo pkill -f node

6. Verify the audit logger methods exist:
   grep -n "logReturnCreate" EventAuditLogger.js
   grep -n "logReturnCreate" ProductionEventAuditLogger.js

7. Start the server:
   nohup node server.js > server.log 2>&1 &

8. Check server status:
   ps aux | grep "node server.js" | grep -v grep

9. Check server log for errors:
   tail -20 server.log

10. Test the server:
    curl -s http://localhost:5000/api/health
    curl -s 'http://localhost:5000/api/audit-logs?page=1&limit=3'

EXPECTED RESULT:
================
- No more "logReturnCreate is not a function" errors
- No more "Unknown column 'al.resource'" errors  
- Server starts without module errors
- Audit logs endpoint works properly
- Return creation audit logging works

If you still get errors, the audit logger files may not have updated properly.
In that case, we can manually upload them again using SCP.
