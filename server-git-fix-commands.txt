MANUAL COMMANDS TO FIX GIT PULL CONFLICT ON SERVER
==================================================

SSH to your server:
ssh -i "C:\Users\Admin\e2c.pem" ubuntu@54.169.107.64

Then run these commands one by one:

1. Go to project directory:
cd ~/inventoryfullstack

2. Backup your current .env.production:
cp .env.production .env.production.backup

3. Stash local changes:
git stash push -m "Backup local env changes"

4. Pull latest changes:
git pull origin main

5. Restore your .env.production if needed:
cp .env.production.backup .env.production

6. Check git status:
git status

7. Apply database migration:
sudo mysql inventory_db < fix-audit-logs-schema.sql

8. Restart server:
pm2 restart server

EXPLANATION:
============
- Your server has local changes to .env.production (probably API URLs)
- GitHub has a different version of .env.production  
- We stash your changes, pull updates, then restore your settings
- This preserves your server configuration while getting code updates

ALTERNATIVE QUICK FIX:
=====================
If you want to keep your current .env.production exactly as is:

git checkout -- .env.production
git pull origin main

This will discard the conflicting file and pull updates.
